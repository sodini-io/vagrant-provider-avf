Quickstart

From install to shell in three clean steps.

The provider plugin and the box are separate artifacts. Install the gem once, then add the Linux box you want.

01

Install the provider

vagrant plugin install vagrant-provider-avf
02

Add a box

vagrant box add sodini-io/ubuntu-24.04-arm64 --provider avf --box-version 0.1.0
03

Bring it up

vagrant up --provider avf
vagrant ssh

Why this exists

Not a VM platform. A sharp tool.

The point of this project is not to turn Vagrant into another sprawling virtualization layer. The point is to make Vagrant feel useful again on Apple Silicon with a native runtime, a curated Linux story, and a support surface small enough to understand.

That is why the release line is intentionally narrow: Linux only, headless only, Apple Silicon only, with no bridged networking, no GUI path, and no snapshot surface pretending to be done before it is ready.

Native runtime Signed local helper on Apple `Virtualization.framework`
SSH Localhost forwarding with honest `ssh_info`
Shared folders Read/write Linux `virtiofs` mounts

Examples

Simple first. Tuned second.

Minimal Ubuntu
Vagrant.configure("2") do |config|
  config.vm.box = "sodini-io/ubuntu-24.04-arm64"
  config.vm.box_check_update = false

  config.vm.provider :avf do |avf|
    avf.cpus = 2
    avf.memory_mb = 2048
    avf.disk_gb = 8
    avf.headless = true
  end
end
Shared folders and tuning
Vagrant.configure("2") do |config|
  config.vm.box = "sodini-io/ubuntu-24.04-arm64"
  config.vm.box_check_update = false

  config.vm.synced_folder ".", "/vagrant"
  config.vm.synced_folder "examples", "/home/vagrant/examples", type: :avf_virtiofs

  config.vm.provider :avf do |avf|
    avf.guest = :linux
    avf.cpus = 4
    avf.memory_mb = 4096
    avf.disk_gb = 16
    avf.headless = true
  end
end

What works today

Support surface with real edges.

Included

  • `up`, `halt`, `destroy`
  • `ssh_info` and `vagrant ssh`
  • localhost SSH forwarding
  • stable forwarded port reuse
  • Linux shared folders through `virtiofs`
  • configurable `cpus`, `memory_mb`, `disk_gb`, `headless`

Deliberately out

  • bridged networking
  • snapshots
  • GUI paths
  • BSD guest support
  • x86 emulation
  • feature sprawl for the sake of symmetry
